# SaaS Case Study Page

## Best For

- customer story page
- B2B credibility visual
- case study website screenshot

## Prompt (EN)

```text
Goal: Create a premium SaaS case-study page screenshot that communicates proof, clarity, and enterprise trust.
Asset: Desktop customer-story page for a B2B software website.
Scene: Editorial case-study layout with hero statement, customer logo area, metrics highlights, workflow screenshots, and testimonial blocks.
Subject: One well-structured case-study web page focused on measurable outcomes and polished software storytelling.
Composition: Tall desktop page, centered and magazine-like, clear vertical reading rhythm, balanced content modules, not left-heavy.
Lighting: Clean UI rendering with subtle image treatment inside screenshots and soft premium page depth.
Materials and styling: Enterprise storytelling design, muted navy or slate palette, metric cards, dashboard snippets, trust-building visual hierarchy.
Constraints: No clutter, no fake busy data overload, no unreadable charts, no broken layout, no watermark.
```

## 提示词（中文）

```text
目标：生成一张高级 SaaS 客户案例页截图，让页面既传达结果证明，也具备企业级可信度。
资产类型：B2B 软件官网的客户故事 / case study 页面桌面端截图。
场景：编辑式案例版式，包含主标题、客户 logo 区、结果数据卡、流程截图和引用证言模块。
主体：一个围绕“可量化结果 + 软件价值叙事”展开的完整案例页面。
构图：纵向桌面页面，整体居中，阅读节奏像杂志专题，内容模块平衡，不要左重右轻。
光线：清晰 UI 渲染，截图内有轻微图像处理和柔和层级感。
材质与风格：企业级叙事设计、克制海军蓝或石板灰配色、数据卡、dashboard 片段、强调信任的层级组织。
约束：不要杂乱，不要假装忙碌的数据堆叠，不要不可读图表，不要布局损坏，不要水印。
```
